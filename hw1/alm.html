Application Lifecycle Management

ALM is the discipline of managing a software application from initial concept through retirement, integrating governance, development, testing, and operations into one repeatable process.

ALM bridges the gap between business stakeholders and engineering teams by providing shared processes and tooling across every stage of delivery. In the Microsoft ecosystem it is typically practiced with ADO or GitHub, both of which cover the full Dev → Test → UAT → Production promotion chain.

  • Governance — establishes branching strategies, release gates, security baselines, and naming conventions
  • Planning — backlog creation, sprint planning, and work estimation using Azure Boards or GitHub Projects
  • Development — coding, peer review via pull requests, and tracking changes in source control
  • Testing — automated unit, integration, and UAT tests enforced as pipeline quality gates
  • Deployment — promoting artifacts through environments via CI/CD pipelines with required approvals
  • Operations — monitoring health and performance in production and feeding observations back into the backlog
  • Retirement — decommissioning an application including data archival, user communication, and resource cleanup
